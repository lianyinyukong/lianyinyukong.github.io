第二章──數據操作 (Data Manipulation)

1. 電腦架構 (Computer Architecture)
CPU 是電腦的核心，用來執行所有指令與運算。它主要由以下幾個部分組成：

1.算術／邏輯單元（Arithmetic/Logic Unit, ALU）

  • 負責執行數學運算（例如加減乘除）
  • 處理邏輯運算（例如 AND、OR、NOT、比較大小等）


2. 控制單元（Control Unit, CU）

  • 負責解讀程式指令，控制其他部件的運作
  • 指導資料在記憶體、暫存器與輸出入裝置之間的流動


3. 暫存器（Registers）
暫存器是 CPU 內部的高速小型記憶體，用來暫時儲存資料或指令

  • 一般用途暫存器（General-purpose registers）：
    用來存放運算中間結果或臨時資料

  • 特殊用途暫存器（Special-purpose registers）：
    用來執行特定任務，例如:

  • 程式計數器（Program Counter, PC）：記錄下一條要執行的指令位置
  • 指令暫存器（Instruction Register, IR）：儲存目前正在執行的指令
  • 狀態暫存器（Status Register）：紀錄 CPU 的運算狀態
    （例如正負號、是否溢出等）


    2. 匯流排（Bus）
    匯流排是資料傳輸的通道，用來連接 CPU、記憶體和其他裝置。
    主要分為三類：

    • 控制匯流排（Control Bus）：傳送控制訊號（例如讀、寫、啟動）
    • 資料匯流排（Data Bus）：傳送資料
    • 位址匯流排（Address Bus）：傳送記憶體位


    3.主機板（Motherboard）
    它負責提供電力與連接通道，讓所有硬體能協同運作
    主機板是整台電腦的主要電路板，所有主要元件都安裝在上面
    （CPU、記憶體、匯流排、I/O 介面等）

    4. 機器指令（Machine Instruction）
    一條以 位元組成的指令（bit pattern），能被 CPU 辨識並執行
    換句話說，就是 CPU 真正「看得懂」的命令
    每一條機器指令會告訴 CPU 要做什麼事，例如：

    • 加法（ADD）
    • 資料載入（LOAD）
    • 資料儲存（STORE）
    • 跳躍（JUMP）等


    5. 機器語言（Machine Language）
    就是一台電腦所能辨識的所有機器指令的集合
    也就是「那台 CPU 的語言」
    每一種 CPU（例如 Intel、ARM、MIPS）都有自己的機器語言
    人類通常不直接寫機器語言，而是用組合語言（Assembly Language） 代替，然後再由組譯器（Assembler）轉成機器碼。


    #補充 組合語言（Assembly Language）
    是一種 接近機器語言（Machine Language）的低階程式語言，
    它用「英文縮寫符號」來代表機器指令中的二進位碼

    人類幾乎不可能直接看懂或維護這種程式。
    所以工程師發明了「組合語言」——
    讓這串位元有「可讀」的文字版本：

    這兩行其實是一模一樣的指令!
    上面那行是電腦看的（二進位碼)
    下面那行是人看的（組合語言）

    6. 機器語言設計哲學 (Machine Language Philosophies)
    也就是電腦在設計 CPU 指令集時，採用的兩種不同策略：
    RISC（精簡指令集） 和 CISC（複雜指令集）


    • 精簡指令集電腦（Reduced Instruction Set Computing，RISC）
      使用「少量、簡單、高效率」的指令，讓執行速度更快

      代表架構：
      ARM（幾乎所有智慧型手機都用它）
      PowerPC（早期 Apple 電腦、遊戲主機如 PS3 用過）


    • 複雜指令集電腦（Complex Instruction Set Computing，CISC）
      使用「指令多、功能強」的設計，讓程式更簡短、更方便

      代表架構：
      Intel x86 系列（用在絕大多數桌上型與筆電 CPU）
      AMD（與 Intel 相容）

    7. 機器指令的三大基本種類（Machine Instruction Types）
    CPU 所能執行的所有指令，大致可以分成三大類：

    1. 資料傳輸指令(Data Transfer Instructions)：
    用來在不同位置之間「搬移資料」
    這類指令不改變資料內容，只是把資料從一個地方複製到另一個地方
    例如：

    • 從記憶體載入到暫存器（Load）
    • 從暫存器儲存回記憶體（Store）
    • 暫存器之間互相傳送（Move）


    2. 算術／邏輯指令（Arithmetic / Logic Instructions）
    使用現有的位元資料（bit patterns）進行運算，產生新的結果
    這些指令讓 CPU 能進行數學計算或邏輯比較
    例如：

    • 算術運算：加（ADD）、減（SUB）、乘（MUL）、除（DIV）
    • 邏輯運算：AND、OR、NOT、XOR
    • 比較運算：COMPARE（用來判斷大小、是否相等）


    3. 控制指令（Control Instructions）
    用來改變程式執行流程或控制指令執行順序
    它們讓 CPU 能夠根據條件跳躍、重複或結束程式
    例如：

    • 跳躍（Jump / Branch）：改變下一條要執行的指令位址
    • 條件跳躍（Conditional Jump）：根據比較結果決定是否跳轉
    • 副程式呼叫（Call / Return）：進入或返回子程式
    • 停止（Halt）：停止執行


    8. 機器指令的組成（Parts of a Machine Instruction）
    每一條 機器指令（Machine Instruction） 通常由兩個主要部分組成：

    1. 操作碼（Op-code, Operation Code）
    指出「要執行哪一種操作」
    操作碼是指令中的動作部分，告訴 CPU 要做什麼事
    例如：

    • 加法（ADD）
    • 載入（LOAD）
    • 儲存（STORE）
    • 跳躍（JUMP）


    2. 運算元（Operand）
    提供執行該操作所需的額外資訊
    運算元通常指定：

    • 要操作的資料來源或目的地（例如暫存器或記憶體位址）
    • 或是常數值（immediate value）

    但要注意 運算元的「解釋方式」會依照操作碼的不同而改變!!!

    9. 程式執行（Program Execution）
    也就是 CPU 是怎麼一步步執行程式碼的
    CPU 在執行程式的時候，
    是透過一個不斷重複的循環來完成的：
    取指（Fetch）→ 解碼（Decode）→ 執行（Execute）
    這個循環叫做 Machine Cycle（機器週期） 或 Instruction Cycle（指令週期）

    1. 兩個關鍵暫存器（Special-Purpose Registers）
    CPU 內部有許多「暫存器（Register）」，其中有兩個是專門控制程式執行流程的

    11. CPU 如何與外部裝置溝通（Communicating with Other Devices）
    CPU 並不是直接控制每個硬體（印表機、滑鼠、螢幕等），
    而是透過「控制器（Controller）」這種中介裝置來完成溝通的

    1. Controller（控制器）
    控制器是一種「中間人」，負責協調 CPU 與外部裝置之間的資料交換
    它的主要功能是：

    • 將 CPU 的指令轉成外設（例如滑鼠、硬碟）聽得懂的訊號
    • 再把外設回傳的資料轉回 CPU 能理解的格式

    例如：

    • 鍵盤控制器（Keyboard Controller）
    • 顯示卡（Display Controller / GPU）
    • 磁碟控制器（Disk Controller）


    2. Port（連接埠）
    「Port」就是裝置連接到電腦的實體或邏輯介面

    例子：

    • USB 埠
    • HDMI 埠
    • Ethernet（網路埠）

    換句話說，「Port」是「接點」，「Controller」是「翻譯官」


    3. Memory-mapped I/O（記憶體對映輸入輸出）
    這是一種讓 CPU 把「外部裝置」當成記憶體單元來溝通的方式

    1. Parallel Communication（平行通訊）
    同時使用多條傳輸路徑（wires），一次傳送多個位元（bits）


    2. Serial Communication（序列通訊）
    使用單一路徑（wire），一次傳送一個位元（bit）


    12. 資料傳輸速率（Data Communication Rates）


    13. 程式設計中的資料操作（Programming Data Manipulation）
    程式語言幫助我們「抽象化」硬體的細節：
    使用者在寫程式時，不需要知道電腦底層是怎麼運作的


    14. 其他電腦架構（Other Architectures）
    重點是如何提高 處理效率與資料吞吐量（throughput）

    總結：

    Pipelining 提升單核效率，Parallel Processing 提升多核效率；
    架構不同（SISD / SIMD / MIMD）決定同時處理資料或程式的能力
